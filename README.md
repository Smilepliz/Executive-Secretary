# Руководство пользователя

Этот проект — демонстрационный прототип редакционного процесса статьи.  
Ваша задача как пользователя: двигать статью по этапам, выполнять доступные действия и отслеживать историю переходов.

## Что показывает интерфейс

- **Сценарии демонстрации** — быстрый выбор стартового состояния статьи.
- **Этапы процесса** — визуальная шкала этапов от антиплагиата до публикации.
- **Текущий контекст** — текущий этап и текущий статус статьи.
- **Доступные действия** — только те действия, которые сейчас разрешены.
- **История** — журнал изменений статусов и источников переходов.

## Основная логика работы

- В каждый момент времени статья имеет **один текущий статус**.
- Каждый статус принадлежит определенному этапу процесса.
- Нажимать можно только те действия, которые доступны для текущего статуса.
- После действия статус обновляется, и интерфейс пересчитывает:
  - текущий этап;
  - список доступных действий;
  - историю.

## Как работать в прототипе

1. Выберите нужный сценарий в блоке **Сценарии демонстрации**.
2. Посмотрите блок **Текущий контекст**, чтобы понять, где находится статья.
3. Выполните одно из действий в блоке **Доступные действия**.
4. Проверьте изменения в блоке **История**.
5. Повторяйте, пока не дойдете до нужного результата (публикация, возврат на доработку, отклонение и т.д.).

## Работа с этапами (демо-режим)

- Клик по любому этапу открывает окно выбора статуса внутри выбранного этапа.
- Это ручной переход для проверки разных состояний в интерфейсе.
- После подтверждения новый статус применяется сразу, а в истории появляется запись.

Важно: это демонстрационная возможность, чтобы быстро тестировать ветки процесса.

## Как читать историю

- История показывает переходы от нового к старому (сверху свежие записи).
- По записи видно:
  - какое действие выполнено;
  - из какого статуса в какой был переход;
  - источник перехода.
- Источник `system` означает автоматический переход, не ручной клик пользователя.

## Типовой сценарий прохождения статьи

Обычно путь выглядит так:

1. Ожидает запуска проверки
2. Проверка на антиплагиат идет
3. Антиплагиат пройден
4. Переход к этапу рецензентов
5. Назначение рецензентов
6. Рецензирование
7. Один из исходов:
   - запрос правок автору;
   - принятие статьи;
   - отклонение статьи
8. Подготовка к публикации
9. Публикация

## Что делать, если кнопок действий нет

Если в блоке **Доступные действия** нет кнопок:

- проверьте текущий статус;
- переключите сценарий демонстрации;
- используйте клик по этапу для ручного перехода в другой статус (демо-режим).
